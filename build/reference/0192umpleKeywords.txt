<h2>Here is the list of all keywords in the Umple programming language. They cannot be used as identifiers.</h2>

<li><span style="color:red; font-weight: bold">abstract:</span> Defines a class or method as abstract. <a href="AbstractMethods.html"><i>See Abstract Methods</i></a></li>
<li><span style="color:red; font-weight: bold">activate:</span>  Denotes the trace case to be activated.</li>
<li><span style="color:red; font-weight: bold">active:</span> Defines an object that, when initialized, runs its own thread. <a href="ActiveObjects.html"><i>See Active Objects</i></a></li>
<li><span style="color:red; font-weight: bold">add:</span> Indicates that the tracing should occur when an association link is being added. <a href="TracingAssociations.html"><i>See Tracing Associations</i></a></li>

<li><span style="color:red; font-weight: bold">after:</span> (State Machine) Triggers a transition after a given amount of time. <a href="TimedTransitions.html"><i></i>See Timed Transitions</a></li>
<li><span style="color:red; font-weight: bold">after every:</span> (State Machine) Continuously triggers a transition at a specified time intervals while object remains in the state. <a href="TimedTransitions.html"><i>See Timed Transitions</i></a></li>
<li><span style="color:red; font-weight: bold">all:</span> This indicates that code should be injected in every method that matches the specified pattern. <a href=""><i></i></a></li>
<li><span style="color:red; font-weight: bold">allow:</span> Subsequent to a <i>strictness</i> statement, this denotes an error that should be considered as a warning. <a href="StrictnessDirective.html"><i>See Strictness Directive.</i></a></li>
<li><span style="color:red; font-weight: bold">and:</span> And Boolean operator.</li>
<li><span style="color:red; font-weight: bold">around:</span> Indicates that some code will be inserted around a code block  <a href="AroundStatement.html"><i></i></a></li>
<li><span style="color:red; font-weight: bold">around_proceed:</span> Denotes where the original block of code around which code would be inserted should be placed.  <a href="AroundStatement.html"><i>See Around Statement</i></a></li>
<li><span style="color:red; font-weight: bold">as:</span> Declares a function or a state machine alias subsequent to the function or state machine definition. This is optional and should be followed by an alias name.</li>
<li><span style="color:red; font-weight: bold">assertAttribute:</span> Asserts attribute's presence in generated code. <a href="BasicTestcase.html"><i>See Basic Testcase</i></a></li>
<li><span style="color:red; font-weight: bold">assertEqual:</span> Asserts that two objects are equal. <a href="BasicTestcase.html"><i>See Basic Testcase</i></a></li>
<li><span style="color:red; font-weight: bold">assertFalse:</span> Asserts that condition is false. <a href="BasicTestcase.html"><i>See Basic Testcase</i></a></li>
<li><span style="color:red; font-weight: bold">assertMethod:</span> Asserts methods's presence in generated code. <a href="BasicTestcase.html"><i>See Basic Testcase</i></a></li>
<li><span style="color:red; font-weight: bold">assertNull:</span> Asserts that object is null. <a href="BasicTestcase.html"><i>See Basic Testcase</i></a></li>
<li><span style="color:red; font-weight: bold">assertTrue:</span> Asserts that condition is true. <a href="BasicTestcase.html"><i>See Basic Testcase</i></a></li>
<li><span style="color:red; font-weight: bold">association:</span> Defines an association between two classes, separate from the two classes. <a href="IndependentlyDefinedAssociations.html"><i>See Independently Defined Associations</i></a></li>
<li><span style="color:red; font-weight: bold">association Class:</span> Defines a class to manage many to many associations. <a href="AssociationClassDefinition.html"><i>See association class definition</i></a></li>
<li><span style="color:red; font-weight: bold">attribute/attr:</span> TBD <a href=""><i></i></a></li>
<li><span style="color:red; font-weight: bold">attributes:</span> Denotes a set new of attributes to be used. </li>
<li><span style="color:red; font-weight: bold">autounique:</span> Automatically assigns a unique value to an attribute each time a class is instantiated. <a href="UniqueAttributes.html"><i>See unique attributes</i></a></li>
<li><span style="color:red; font-weight: bold">before:</span> Requests that code should be run once a few Umple-defined actions are completed. <a href="BeforeandAfterStatements.html"><i>See Before and After Statements</i></a></li>
<li><span style="color:red; font-weight: bold">between:</span> Defines the associated classes. </li>
<li><span style="color:red; font-weight: bold">cardinality:</span> TBD <a href=""><i></i></a></li>

<li><span style="color:red; font-weight: bold">const:</span> Defines a new constant. <a href="AttributeDefinition.html"><i></i>See Attribute Definition</a></li>
<li><span style="color:red; font-weight: bold">custom:</span> This indicates that code should be injected in every custom method that matches the specified pattern. <a href=""><i></i></a></li>
<li><span style="color:red; font-weight: bold">deactivate:</span> Indicates the trace case to deactivate.</li>
<li><span style="color:red; font-weight: bold">depend:</span> Denotes a dependency on another class, interface or namespace. <a href="Dependclause.html"><i>See Depend Clause</i></a></li>
<li><span style="color:red; font-weight: bold">disable:</span> This prevents warning(s) matching a provided expression to be raised. </li>
